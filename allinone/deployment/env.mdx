## 配置来源

allinone 按以下优先级加载配置：

1. **环境变量**——优先级最高，覆盖配置文件
2. **配置文件**——路径由 `MYAPP_CONFIG_PATH` 指定

所有环境变量使用 `MYAPP_` 前缀。

---

## 配置项

### 服务器

| 变量 | 默认值 | 说明 |
|---|---|---|
| `MYAPP_ANCLAX_PORT` | `2910` | HTTP 服务监听端口 |
| `MYAPP_ANCLAX_ENV` | `development` | 运行环境，生产环境设为 `production` |
| `MYAPP_CONFIG_PATH` | — | 配置文件路径（可选）|

### 数据库

| 变量 | 默认值 | 说明 |
|---|---|---|
| `MYAPP_ANCLAX_PG_DSN` | `postgres://postgres:postgres@localhost:5432/postgres` | PostgreSQL 连接字符串 |

### 认证

| 变量 | 默认值 | 说明 |
|---|---|---|
| `MYAPP_ANCLAX_JWT_SECRET` | — | 签发和验证 Access Token 的密钥，**生产环境必填** |
| `MYAPP_ANCLAX_JWT_EXPIRY` | `15m` | Access Token 有效期（如 `15m`、`1h`）|
| `MYAPP_ANCLAX_REFRESH_SECRET` | — | 签发 Refresh Token 的密钥，**生产环境必填** |
| `MYAPP_ANCLAX_REFRESH_EXPIRY` | `7d` | Refresh Token 有效期 |

---

## 本地开发

不使用 Docker 时，手动导出最小配置：

```bash
export MYAPP_ANCLAX_PORT=2910
export MYAPP_ANCLAX_PG_DSN=postgres://postgres:postgres@localhost:5432/postgres
export MYAPP_ANCLAX_JWT_SECRET=dev-secret-change-in-prod
export MYAPP_ANCLAX_REFRESH_SECRET=dev-refresh-secret-change-in-prod
go run cmd/main.go
```

## Docker Compose 配置

在 `docker-compose.yaml` 的 `environment` 块中设置：

```yaml
services:
  dev:
    environment:
      MYAPP_ANCLAX_PORT: "2910"
      MYAPP_ANCLAX_PG_DSN: postgres://postgres:postgres@db:5432/postgres
      MYAPP_ANCLAX_JWT_SECRET: dev-secret
      MYAPP_ANCLAX_REFRESH_SECRET: dev-refresh-secret
```

生产环境推荐使用 `.env` 文件：

```yaml
services:
  app:
    env_file:
      - .env.production
```

<Warning>
  不要将真实密钥提交到版本控制。使用独立的 `.env.production` 文件（已加入 `.gitignore`），或通过 CI/CD 平台的密钥管理功能注入。
</Warning>

## 前端连接配置

ido 前端通过 `VITE_API_BASE_URL` 连接到 allinone，在 `todo/.env` 中配置：

```bash
VITE_API_BASE_URL=http://localhost:2910/api/v1
```

部署前端到生产环境时，将此地址改为你的生产 API 地址。
