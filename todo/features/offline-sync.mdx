## 工作原理

ido 是离线优先的应用。断网时，所有写操作（新建、修改、删除）不会丢失，而是：

1. **立即更新界面**——操作效果即时可见，无需等待网络。
2. **保存到本地队列**——操作被序列化后存入 `localStorage`，页面刷新也不会丢失。
3. **联网后自动同步**——恢复网络时，队列中的操作按顺序发送到服务器。

## 同步状态指示

页面右上角有一个**同步状态**徽章。当本地队列中有待同步的操作时，徽章会显示提示。联网同步完成后徽章消失。

## 操作队列

每个待同步操作包含以下信息：

| 类型 | 说明 |
|---|---|
| `create` | 新建任务，含临时 ID 和标题 |
| `update` | 修改任务，含任务 ID 和变更字段（标题、桶、完成状态）|
| `delete` | 删除任务，含任务 ID |

操作顺序严格保留。如果某个操作同步失败，后续操作暂停，等待下次重试，以保证数据一致性。

## 临时 ID

断网时新建的任务会获得一个临时 ID（如 `temp-1234567890`）。如果你在同步前又对该任务进行了修改或删除，这些操作同样使用临时 ID 记录在队列中。

当新建操作同步成功后，服务器返回真实 ID，队列中所有引用该临时 ID 的操作会自动替换为真实 ID，再继续执行后续操作。

## 注意事项

<Warning>
  如果你在离线时删除了一个任务，之后联网同步时该删除操作会发送到服务器，任务将被永久删除，无法恢复。
</Warning>

<Note>
  同步过程中如果遇到网络错误，操作保留在队列中，下次联网时继续重试。只有服务器明确返回错误时，操作才会被丢弃（目前实现中会停止队列处理，等待手动重试）。
</Note>
